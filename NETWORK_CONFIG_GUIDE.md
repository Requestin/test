# Руководство по сетевым настройкам для FlashVSR

## Вопросы провайдера и ответы

### 1. Требуется ли интернет-канал с использованием NSX Edge?

**Ответ: ДА** ✅

**Почему:** Нужен доступ в интернет для:
- Скачивания репозиториев с GitHub (FlashVSR, Block-Sparse-Attention)
- Скачивания весов модели с Hugging Face (через Git LFS)
- Установки Python-пакетов через pip

**Что указать:**
- **Количество IP-адресов:** `1` (достаточно одного IP для доступа к серверу)
- **Скорость интернет-канала:** `100 Мбит/с` или выше (рекомендуется `100-1000 Мбит/с`)

> **Примечание:** 
> - Если провайдер предлагает варианты скорости (например, 100/500/1000 Мбит/с), можно выбрать средний вариант (500 Мбит/с), если это не сильно влияет на стоимость
> - Для скачивания весов модели через Git LFS может потребоваться время, поэтому лучше иметь достаточную скорость

---

### 2. Требуется ли интернет-канал директовой подсетью?

**Ответ: НЕТ** ❌ (в большинстве случаев)

**Почему:** 
- Директовая подсеть обычно нужна для публичных IP-адресов без NAT
- Для вашей задачи достаточно обычного интернет-канала через NSX Edge
- Публичный IP обычно предоставляется автоматически через NSX Edge

**Когда может понадобиться директовая подсеть:**
- Если нужен статический публичный IP-адрес
- Если планируется размещение веб-сервисов с внешним доступом
- Если требуются специфические сетевые настройки

**Для FlashVSR это не требуется**, так как:
- Доступ к серверу будет через SSH (обычно работает через NAT)
- Веб-интерфейсы (JupyterLab, WebDesktop) обычно доступны через VPN или специальный портал провайдера
- Не требуется размещение публичных сервисов

## Рекомендуемая конфигурация

### Минимальная конфигурация:
- ✅ **NSX Edge:** ДА
- ✅ **Количество IP:** 1
- ✅ **Скорость интернет-канала:** 100 Мбит/с (минимум)
- ❌ **Директовая подсеть:** НЕТ

### Оптимальная конфигурация:
- ✅ **NSX Edge:** ДА
- ✅ **Количество IP:** 1
- ✅ **Скорость интернет-канала:** 500-1000 Мбит/с (для быстрой загрузки весов модели)
- ❌ **Директовая подсеть:** НЕТ

## Дополнительные вопросы, которые стоит уточнить

Если провайдер предоставляет возможность выбора, уточните:

1. **Тип доступа:**
   - Как будет осуществляться доступ к серверу? (SSH, VPN, веб-портал)
   - Нужен ли статический IP-адрес?

2. **Ограничения трафика:**
   - Есть ли лимиты на интернет-трафик?
   - Включен ли трафик в стоимость или оплачивается отдельно?

3. **Порты и доступность:**
   - Будут ли доступны веб-интерфейсы (JupyterLab, WebDesktop) через интернет или только через VPN?
   - Нужно ли открывать дополнительные порты?

## Почему важна скорость интернет-канала

Для FlashVSR потребуется скачать:
- Репозитории с GitHub (несколько сотен МБ)
- Веса модели с Hugging Face через Git LFS (может быть несколько ГБ)
- Python-пакеты через pip (зависимости проекта)

**Примерное время загрузки:**
- При 100 Мбит/с: ~1-2 часа для всех данных
- При 500 Мбит/с: ~15-30 минут
- При 1000 Мбит/с: ~10-15 минут

Рекомендуется выбрать скорость, которая позволит быстро скачать все необходимое, но не переплачивать за избыточную скорость.
